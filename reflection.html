A responsive web design is designing of a web application taking into account dynamic changes in the size, orientation, and devices. These parameters may change while the user is using the application. For example, a user may be using the application full screen on a desktop, but then decides to make the window smaller. If that happens, the developer should take into account how the application should look at that point to make sure it is still readable. Usually, dynamic web design can be accomplished by using break points in CSS. We can define the break points with parameters such as maximum and minimum screen sizes, etc. Then, the developer can make the application presentable to end-users by doing CSS for each case in a clear manner. This way, the web application can now be presented on small devices such as smartphones and desktops, etc. For examples, on a large enough window, I may want to display a group of elements horizontally, but on smaller one, I may decide to align them vertically.
Responsive web designs relate to web technologies because it is obviously relevant in a way that web applications should be presented in a way that it can handle the dynamic changes in parameters such as sizes of the windows, thus making it easier for the user to use. Again, for now, I can think of developing application with responsive design in CSS, and as I do more projects, there may be times I need to use javascript to handle the way web application is presented. For example, on a large enough screen, I would assume that the user will have the mouse, so I could add on hover behaviors on elements such as change in color on hover, but on smaller one, I could disable that. Another example is, I may want to add some animations to the application for presentation on large enough screen using javascript, but on smaller one, I could just disable it and put an image instead.
